Section 01

Executive Summary

AstraWatch is a real-time space tracking web application that brings live orbital data to the browser in a visually immersive format. The application enables users to track the International Space Station (ISS) on an interactive 3D globe, visualize hundreds of active satellites in orbit, and explore tonight's planetary sky — all personalized to their current location.

Built entirely with modern web technologies (React 19, Vite 5, Tailwind CSS v4), deployed on Vercel with continuous delivery from GitHub, and powered by free, publicly accessible astronomy APIs with graceful fallbacks — no mandatory API keys, no backend server, no installation required.

The interface follows a deep-space mission control aesthetic with glassmorphism panels, animated star fields, and live telemetry readouts — designed to make orbital data accessible to anyone with a browser.

Section 03

System Architecture

AstraWatch follows a client-side Single Page Application (SPA) architecture. All data fetching, orbital computation, and rendering happen entirely in the browser — there is no backend server or database.

┌─────────────────────────────────────────────────────────┐
│                        Browser                          │
│                                                         │
│  ┌──────────────┐    ┌──────────────┐    ┌───────────┐  │
│  │  React 19    │    │  Globe.GL    │    │Canvas API │  │
│  │  Component   │───▶│  (Three.js)  │    │Star Field │  │
│  │  Tree        │    │  3D Renderer │    │           │  │
│  └──────┬───────┘    └──────────────┘    └───────────┘  │
│         │                                               │
│  ┌──────▼──────────────────────────────────┐            │
│  │          Custom React Hooks             │            │
│  │  useISS() │ useSatellites() │ usePlanets()           │
│  └──────┬────────────┬──────────────┬──────┘            │
└─────────┼────────────┼──────────────┼───────────────────┘
          │            │              │
          ▼            ▼              ▼
   WhereTheISS    CelesTrak       Astronomy API
   (ISS Position) (TLE Data)     (Planet Positions)
                               OR Meeus fallback

Data Flow

  1. On mount: All three hooks fire simultaneously — useISS fetches ISS position, useSatellites fetches TLE data and propagates satellite positions, usePlanets computes/fetches planet data.
  2. Every 5 seconds: useISS re-polls the ISS API and appends the new position to the trail buffer (circular buffer, last 30 points).
  3. Every 10 seconds: useSatellites re-runs SGP4 propagation on cached TLEs to update satellite dot positions without re-fetching.
  4. On data change: React re-renders only affected components; Globe.GL layers update in place without reinitializing the WebGL context.

Section 05

Feature Breakdown

5.1 — Interactive 3D Globe

Powered by Globe.GL (Three.js wrapper) with GPU-accelerated WebGL rendering. Features a photorealistic Blue Marble Earth texture, topographic bump map, and a night sky background panorama.

  • ISS Marker — cyan dot, size 1.5, altitude = orbital height ÷ Earth radius
  • Satellite Cloud — up to 200 purple dots from live TLE propagation
  • ISS Trail — 30 consecutive arc segments with opacity fade showing recent path
  • Camera — smooth 1-second fly-to animation on each ISS position update
  • Controls — auto-rotate at 0.5 rpm, momentum damping, zoom & pan

5.2 — ISS Live Telemetry Panel

Left sidebar with a 2×3 stat grid, refreshed every 5 seconds:

5.3 — Satellite Visualization

TLE data is fetched from CelesTrak's "visual" group on mount, parsed into satellite.js satrec objects, and propagated via SGP4 to compute current geographic positions. Re-propagated every 10 seconds using cached TLEs. Hardcoded fallback TLE set (ISS + 5 Starlinks) activates silently on CORS/network failure.

5.4 — Night Sky Panel

  • Animated canvas star field — 200 procedurally-placed stars with independent twinkle animation at 60fps
  • Planet cards for all 7 planets: altitude, azimuth, magnitude, constellation, distance, visible/horizon badge, and an animated altitude progress bar
  • Supports live Astronomy API data with automatic Meeus algorithm fallback

5.5 — Geolocation & Reverse Geocoding

Uses the browser navigator.geolocation API to detect the user's position, then reverse-geocodes via Nominatim (OpenStreetMap, no API key) to display a human-readable city name in the navbar. Defaults to New Delhi (28.6°N, 77.2°E) if permission is denied.

Dependency Placement (Critical)

All build-time packages (vite, @vitejs/plugin-react, @tailwindcss/vite, tailwindcss) are in dependencies, not devDependencies. Vercel's production environment only installs dependencies by default, causing build failures if build tools are categorized as dev-only.

Section 10

Challenges & Solutions

⚠ Challenge 1 — Vercel Skipping Build Dependencies

Vercel's production builds skip devDependencies, causing "Cannot find package '@tailwindcss/vite'" errors despite the package being installed locally.

✅ Solution

Moved all build-time packages (vite, @vitejs/plugin-react, @tailwindcss/vite, tailwindcss) from devDependencies to dependencies.

⚠ Challenge 2 — Vite 8 Rolldown Incompatibility

Vite 8 uses an experimental Rust-based bundler (rolldown) that fails with satellite.js@7's WASM pthreads worker (top-level await in iife output format not supported) and with globe.gl's lodash-es transitive imports.

✅ Solution

Downgraded to Vite 5 (stable rollup bundler) and satellite.js@4 (pure JS, same API, no WASM). Both issues resolved instantly.

⚠ Challenge 3 — CelesTrak CORS Policy

CelesTrak does not consistently set Access-Control-Allow-Origin headers, making direct browser TLE fetches unreliable across different networks.

✅ Solution

Implemented a hardcoded FALLBACK_TLE constant (ISS + 5 Starlinks) that activates silently on any network or CORS error. App continues to function with minimal disruption.

⚠ Challenge 4 — ISS Trail Without a Database

The ISS travels ~7.66 km/s — a persistent visual trail requires storing recent positions, but there is no server or database in this purely client-side app.

✅ Solution

Implemented a client-side circular buffer in useISS, capped at 30 positions, passed directly to Globe.GL's arcs layer as sequential start/end point pairs.

⚠ Challenge 5 — CSS Import Order Warning

Tailwind's @import "tailwindcss" was placed before the Google Fonts @import url(), violating the CSS spec rule that all @imports must precede other rules.

✅ Solution

Reordered so Google Fonts @import url() appears first, followed by @import "tailwindcss".

Section 12

Future Scope

Near-term Enhancements

  • Bundle optimization — Code-split Three.js/Globe.GL via dynamic import() to reduce initial load
  • PWA / Service Worker — Cache TLE data and ISS last-known position for offline use
  • ISS Pass Prediction — Use SGP4 to predict when the ISS will pass over the user's location
  • Satellite search — Filter and highlight satellites by name or NORAD ID

Medium-term Features

  • Constellation overlay — Draw constellation lines on the Sky View using star catalog data
  • Moon phase tracker — Lunar phase, rise/set times, and illumination percentage
  • Historical playback — Replay ISS orbital path for a selected date using archived TLEs
  • Space events calendar — Launches, spacewalks, and astronomical events

Long-term Vision

  • AR mode — Device camera + orientation API to overlay satellite positions on the real sky
  • Multi-satellite telemetry — Expand sidebar to any clicked satellite, not just ISS
  • Social sharing — Share ISS pass times via generated links with embedded sky maps
Section 13

Conclusion

AstraWatch successfully demonstrates how modern browser APIs and open-source astronomy data can be combined into a compelling, educational space visualization tool — with no backend infrastructure, no mandatory API keys, and no installation required.

The project showcases:

  • Real-time data integration from multiple free APIs with robust fallback handling
  • WebGL 3D visualization using Globe.GL and Three.js for GPU-accelerated rendering
  • Orbital mechanics in the browser via satellite.js SGP4 propagation from raw TLE data
  • Production-grade deployment on Vercel's global CDN with continuous delivery from GitHub
  • Thoughtful UX design — mission control aesthetic that is visually impressive without sacrificing usability

The application is open-source, publicly deployed, and immediately usable by anyone interested in space — from students learning about orbital mechanics to enthusiasts who want to know exactly where the ISS is right now.
